# 统计口径

## 统计对象

- 院校范围：网站公开接口中标记为 `985`，且招收 `0801` 下属专业的院校。
- 专业范围：`080100`、`080101`、`080102`、`080103`、`080104`、`0801J1`、`0801J7`。
- 去重方式：以院校代码为唯一键；同一院校的不同学院、校区和专业记录合并统计。
- 当前结果：26 所院校、39 条院系或专业记录。

## 字段映射

| 本站字段 | 来源字段或区块 | 当前解释 |
|---|---|---|
| 招生计划 | `enroll_num` | 招生目录或来源数据中的计划人数 |
| 复试人数 | `retest_count` | 来源数据记录的进入复试人数 |
| 拟录取人数 | `enroll_count` | 来源数据记录的拟录取人数 |
| 复试基本线 | `retest_total_score_line` | 专业查询接口中的复试总分要求字段 |
| 聚合复试线 | `score_line` | 来源卡片标为“复试线”的当前值 |
| 复录比 | `rate` | 来源接口给出的复试人数与录取人数比例文本 |
| 研究方向、初试科目 | 详情页“研究方向与初试科目” | 逐方向保留科目组合，不在模板里猜字段 |
| 培养方式、学制、推免说明 | 详情页招生信息区 | 保留来源原文，缺失时展示“待核验” |
| 复试细则 | 详情页“复试细则” | 提取总成绩公式、复试内容、参考书目等公开摘要 |
| 历年分数与复录数据 | 详情页 2023—2026 表格 | 政治、外语、业务课、总分、最高/最低分及录取结构 |

## 缺失值处理

专业查询接口当前值中的 `0`、`null`、`-` 或缺失值可能表示没有采集到数据，而不一定表示真实人数为 0。因此：

1. 只汇总大于 0 的已知值。
2. 同时展示“已知记录数 / 总记录数”。
3. 任一字段覆盖不完整时，该校标记为“部分数据”。
4. 不用缺失值参与差额、报录比或复录比推断。

## 年份边界

专业查询接口本身不带年份；其对应详情页将当前招生计划、复试线、复试人数和录取人数放在 `2026` 列，并提供部分 `2023—2025` 历史记录。因此本站把它描述为“来源页 2026 聚合数据”，而不是“2026 年官网数据”。只有定位到具体年度招生目录、复试名单和拟录取公示后，才能升级为“官网核验”。

## 字段完整度

详情库以参考页为最低字段基线，覆盖：院校标签、院系、专业、培养方式、学制、研究方向、初试科目、招生计划、复试细则、复试线、历年单科成绩、最高/最低分、复试人数、录取人数、复录比、一志愿和调剂录取。本站额外提供 26 所院校横向筛选、逐字段覆盖率、院校官网入口和证据等级。

## 官网核验标准

每所学校至少需要三类原文：

1. 当年硕士招生专业目录或招生计划，用于确认招生人数。
2. 学院复试名单或复试结果，用于统计复试人数。
3. 学院或研究生院拟录取公示，用于统计拟录取人数。

原文需要记录标题、URL、发布日期、适用学院、专业代码、统计年份和名单定位。只有研究生院首页或招生网首页不能直接证明人数。

## 官网字段映射

| 官网原文 | 前端字段 | 空值与冲突处理 |
|---|---|---|
| 招生目录、复试细则中的公开招考计划 | `officialPlanned` | 与含推免总计划分开；口径不明则不填 |
| 明确列出专业代码的复试名单 | `officialRetest` | 合并多个专业且无专业列时只保存证据，不填人数 |
| 明确列出录取专业和结果的公示 | `officialAdmitted` | 分批公示按同一专业累加，并在证据备注说明 |
| 学院专业复试线 | `officialScoreLine` | 学校门类基本线仅作部分证据，不冒充学院线 |
| 标题、URL、日期、范围、状态、备注 | `officialEvidence` | 每项官网值必须有对应证据；找不到原文保留待核验 |

组件只消费稳定的静态数据模型，没有运行时外部请求、表单、权限或持久化状态。官网 URL 使用普通链接输出，不使用 `v-html`。
